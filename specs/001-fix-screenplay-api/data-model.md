# Data Model: Fix Screenplay API Test

**Feature**: 001-fix-screenplay-api  
**Created**: 2026-03-19

## Entities

### Event

Represents an event in the ticketing system. Used for both request and response.

```java
package com.ticketing.models;

public class Event {
    private String id;           // Auto-generated by API
    private String name;         // Required
    private String description;   // Required
    private String date;        // ISO format (YYYY-MM-DD)
    private String venue;       // Required
    private boolean isActive;   // Auto: true on creation
    private int capacity;       // Required, positive integer
}
```

### Request Example

```json
{
  "name": "Concert Test Event",
  "description": "Test concert event",
  "date": "2026-12-31",
  "venue": "Test Arena",
  "capacity": 5000
}
```

### Response Examples

**Create Response (201)**
```json
{
  "id": "evt_123456",
  "name": "Concert Test Event",
  "description": "Test concert event",
  "date": "2026-12-31",
  "venue": "Test Arena",
  "isActive": true,
  "capacity": 5000
}
```

**Update Response (200)**
```json
{
  "id": "evt_123456",
  "name": "Updated Concert Event",
  "description": "Updated description",
  "date": "2026-12-31",
  "venue": "Updated Arena",
  "isActive": true,
  "capacity": 6000
}
```

**Deactivate Response (200)**
```json
{
  "id": "evt_123456",
  "name": "Updated Concert Event",
  "description": "Updated description",
  "date": "2026-12-31",
  "venue": "Updated Arena",
  "isActive": false,
  "capacity": 6000
}
```

## Configuration

| Property | Default | Description |
|----------|---------|-------------|
| baseUrl | http://localhost:50001 | API base URL |

Pass via: `gradle test -DbaseUrl=https://api.example.com`

## API Endpoints

| Method | Endpoint | Body | Status |
|--------|----------|------|--------|
| POST | /events | Event (without id) | 201 |
| GET | /events/{id} | - | 200 |
| PUT | /events/{id} | Event | 200 |
| POST | /events/{id}/deactivate | - | 200 |

## Lifecycle Flow

```
1. POST /events                    → 201, returns Event with id
2. GET /events/{id}               → 200, verify data
3. PUT /events/{id}               → 200, returns updated Event
4. GET /events/{id}               → 200, verify update
5. POST /events/{id}/deactivate   → 200, returns Event with isActive=false
6. GET /events/{id}               → 200, verify isActive=false
```
